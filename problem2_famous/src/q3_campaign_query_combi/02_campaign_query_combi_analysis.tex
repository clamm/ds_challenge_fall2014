Look at visits with orders:
\begin{knitrout}
\definecolor{shadecolor}{rgb}{0.969, 0.969, 0.969}\color{fgcolor}\begin{kframe}
\begin{alltt}
\hlstd{orderLevels} \hlkwb{<-} \hlkwd{names}\hlstd{(}\hlkwd{unlist}\hlstd{(}\hlkwd{sapply}\hlstd{(}\hlkwd{levels}\hlstd{(visitData}\hlopt{$}\hlstd{action),} \hlkwc{FUN}\hlstd{=grep,} \hlkwc{pattern}\hlstd{=}\hlstr{"order"}\hlstd{)))}
\hlstd{orderLevels}
\end{alltt}
\begin{verbatim}
## [1] "[landed order]"        "[landed signup order]"
\end{verbatim}
\begin{alltt}
\hlstd{isOrderIdx} \hlkwb{<-} \hlkwd{sort}\hlstd{(}\hlkwd{union}\hlstd{(}\hlkwd{which}\hlstd{(visitData}\hlopt{$}\hlstd{action} \hlopt{==} \hlstd{orderLevels[}\hlnum{1}\hlstd{]),}
                      \hlkwd{which}\hlstd{(visitData}\hlopt{$}\hlstd{action} \hlopt{==} \hlstd{orderLevels[}\hlnum{2}\hlstd{])))}
\hlstd{totalOrders} \hlkwb{<-} \hlkwd{length}\hlstd{(isOrderIdx)}
\end{alltt}
\end{kframe}
\end{knitrout}

I conclude from the factor levels for \verb+action+ that there is at most 1 order per visit and overall 47348 orders. I cross check with a simple grep on the command line on the unaggregated web data which gives us the same result:

\begin{lstlisting}
$ grep -o order web.log | wc -l
$ 47348
\end{lstlisting}

Add the number of orders per visit as variable to the data frame:
